ADSYNC3 - CONTEXTO PRONTO PARA ENVIAR EM CHAT
=============================================

RESUMO DO PROJETO
- SaaS para gestão financeira de operações de tráfego pago (ADS).
- Stack: Next.js App Router + TypeScript + Supabase (Auth/Postgres/RLS) + TailwindCSS.
- Foco: segurança multiusuário, clareza operacional, cálculo financeiro confiável.

PAPEIS
- dono: visão global da plataforma.
- admin: vê operações próprias + equipe vinculada; gerencia gestores.
- gestor: vê apenas os próprios dados.

ROTAS PRINCIPAIS
- /login
- / (dashboard por papel)
- /operacoes
- /operacao/[id]
- /despesas
- /gestores
- /gestores/[id]
- /convites
- /convite (aceite por token)
- /contabilidade

BANCO (TABELAS CHAVE)
- operacoes (com repasse_percentual por operação)
- lancamentos
- despesas
- profiles (role + nome)
- admin_gestores (vínculo admin -> gestor, com status)
- user_invitations (convites por email com token hash)
- admin_gestor_taxas (overrides administrativos por gestor)

SEGURANÇA
- RLS aplicada nas tabelas de domínio e convites.
- Regra essencial: nunca consultar operação só por id sem escopo de permissão.
- Gestor só acessa ownership próprio.
- Admin acessa equipe via vínculo em admin_gestores.

CALCULO FINANCEIRO BASE
- receita = usd * cotacao_dolar
- custo = facebook + taxa_facebook + taxa_network + taxa_imposto
- lucro = receita - custo
- roi = (lucro / custo) * 100
- repasse = lucro * repasse_percentual / 100
- desconto de despesa = valor * percentual_desconto / 100
